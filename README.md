## 1. 概要 (Overview)
[Manpukuya]は友人（アクティブユーザー4名）を対象とした、「楽しさ」と「健康」を両立させるための食費・健康管理Webアプリケーションです。

単なる記録ツールではなく、ゲーミフィケーションやソーシャル機能を通じて、継続的な利用と生活改善のモチベーションを維持することを目的としています。

## 2. 制作背景・目的 (Motivation)
このプロジェクトを作ろうと思った背景や、解決したかった課題です。

* 友人達が不健康でかつ貯金ができないという課題を抱えており、節約が得意な私に食事を管理して欲しいと頼まれた。(2025/08)
* Line公式アカウントを作り、チャットによって送られた内容をExcelで登録する食費管理サービスを運営し始めた。(2025/08)
* 食費の登録が管理側でしかできず、管理側の負担が大きかった+食事記録場所と食費記録場所が異なるため、利便性が低く、ほぼ知識0の状態からウェブアプリの開発を開始。(2025/08)
* AI(google gemini)を学習メンターとして活用し試行錯誤を繰り返しながらウェブアプリ公開・移行に成功。(2025/10/01)
## 3. 主な機能 (Features)
このプロジェクトの主な機能です。

* 基本機能：食事の登録・集計機能
* ゲーミフィケーション機能：継続利用を促すためのポイント＆ガチャ機能
* コメント機能：食事のフィードバックを含め管理側とユーザーで連絡をするためのコメント機能
* ソーシャル機能：ユーザー間で月別の食事を比較

## 5. 課題と解決策 (Challenges & Solutions)
開発中に直面した課題と、それをどう乗り越えたかです。

* **課題:** サービスを公開したものの、アクティブユーザー（友人）の継続率が低かった。
* **解決策:** 一緒に食事を楽しむお供キャラクターや専用セリフを開放するガチャ機能を追加、食事を登録するとガチャポイントが貯まるので利用意欲を掻き立てることに成功。

* **課題:** 他のユーザーの状況がわからず、自身の食費管理が適切か判断にしくく食費を抑えようというモチベーションが低かった。
* **解決策:** ユーザーの月別の食費をお互いに確認できるページを作成、競争意欲を掻き立てモチベーション向上を図った。

## 6. 今後の展望 (Future Plans)
今後、このプロジェクトをどうしていきたいかです。

* 本来の主旨「楽しく、健康的に食事管理」から逸れないために不用意に機能を複雑化せず、シンプルな運用を続ける。
* ユーザーが飽きないようにガチャのキャラクターの追加等は定期的に実施する予定。
